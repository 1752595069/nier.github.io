git常用命令

  • **+设置用户信息
    • git config –global user.name “字符”
    • git config –global user.email “~~”
  • 查看配置信息
    • git config –list
  • git添加文件
    • git add 文件名

01 获取Git仓库

**（1）获取Git仓库 – 在本地初始化Git仓库 **

**获取Git仓库 – 在本地初始化Git仓库 **

  1. 在任意目录下创建一个空目录 作为本地Git仓库
  2. 进入目录 点击邮件打开 Git bash窗口
  3. 执行命令 git init

（2）获取Git仓库 – 从远程仓库克隆

命令形式： git clone [远程Git仓库地址]

相关概念

版本库 : .git隐藏文件夹 存储很多配置信息

工作区：包含.git文件夹的目录就是工作区 也称为工作目录

暂存区：.git 中的 index文件 临时保存文件目录

02 git工作区中文件的状态

  • unstracked 未跟踪（未被纳入版本控制）
  • tracked 已跟踪 （已修改状态）
    • unmodified 未修改状态
    • modified 已修改状态
    • staged 已暂存状态

**本地仓库操作 **

git status 查看文件状态

git add 将文件的修改存放到暂存区

git reset 将暂存区的文件取消暂存或者是切换到指定版本

git reset –hard 版本号

git commit -m 将暂存区的文件修改提交到版本库

git log 日志文件

03 远程仓库操作

git remote （-v） 查看远程仓库

git remote add 添加远程仓库

git clone 用远程仓库克隆

git pull [short-name] [branch-name] 从远程仓库拉取

  • 注意：如果本地仓库不是从远程仓库克隆 而是本地创建的仓库 并且仓库 中存在文件 此时在从远程仓库拉取文件会报错（fatal:refusing to merge unrelated histories） 加入– —allow-unrelated-histories

git push [remote-name](仓库名 git remote) branch-name 推送到远程仓库

04 分支操作

  • git branch （-r）(-a) 查看分支

    • git branch -r 列出所有远程分支
    • git branch -a 列出所有本地仓库分支和远程仓库分支

  • git branch [name] 创建分支

  • git checkout [name] 切换分支

  • git push [shortName] [name] 推送至远程仓库分支

  • git merge [name] 合并分支

    分支时合并冲突解决（同一个文件 两个分支修改过）

05 标签操作

标签： 指的是某个分支耨个特定时间点的状态

  • git tag 查看已有的标签
  • git tag [name] 创建标签
  • git push [shortName] [name] 将标签推送至远程仓库
  • git checkout - b [branch] [name] 检出标签（需要创建一个新的分支 ）
